# 4.1 — План декомпозиции `main.py`

> Синтез по 6 анализам в [`4.1-analysis/`](4.1-analysis/). `main.py` — 5091 строка.
> Цель: разбить на модули **без изменения поведения**.

## Стратегическое решение (требует подтверждения)

Анализ показал, что задача 4.1 в плане смешивает **две** цели:

1. **Декомпозиция** — разбить `main.py` на модули.
2. **Убрать side-effects при импорте** — отложенная инициализация `Bot`/БД/логов
   (`init_config()`, `setup_logging()`, …).

Цель 2 была обоснована задачей **2.1 (CI)** — чтобы тесты импортировали `main`
без `BOT_TOKEN`/`ENCRYPTION_KEY`. **Но 2.1 отменён** (деплой через Coolify).
Без CI у цели 2 не осталось драйвера, а риск она добавляет большой (меняет
порядок старта, ломает предположения всех тестов и `conftest`).

**Рекомендация: делать только цель 1 — чистую декомпозицию.**
Модули сохраняют свои import-side-effects (как сейчас в `main.py`); важен лишь
порядок импортов. Это кратно снижает риск. Цель 2 — отдельная задача на потом.

## Принципы безопасного извлечения

- **`main.py` остаётся фасадом**: реэкспортит публичные имена (`from config import *`),
  чтобы существующие `main.X` и тесты продолжали работать **без правок**.
- **Разделяемое изменяемое состояние** (`conn`, `cursor`, `_login_attempts`, `apis`,
  `controllers`, `message_states`, `all_groups_timetable_cache`) — доступ только через
  `import module; module.name`, **никогда** `from module import name` (иначе `conftest`
  фикстуры и переприсваивания глобалей ломаются).
- **`conftest.py` правится** для модулей с подменяемым состоянием: `temp_db` →
  `db.conn/db.cursor`, `reset_rate_limit` → `security._login_attempts`.
- Каждый модуль — отдельный коммит, `pytest` зелёный (183 теста) на каждом шаге.

## Главные риски (из анализа)

- Скрытый баг: `LK_CONCURRENCY/DELAY/JITTER` читаются **до** `load_dotenv()`.
  При переносе сохранить точный порядок (env-read до `load_dotenv`).
- `all_groups_timetable_cache` переприсваивается → только доступ через модуль.
- `_alert_admins_parser_broken` использует глобальный `bot` → late import / `set_bot()`.
- Цикл `lk_client ↔ lesson_controller` (`auto_login_user` создаёт контроллер) → late import.
- `aiogram`-хэндлеры на `@dp` → переход на `Router()` + `include_router`.
